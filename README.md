# Incremental Backup Daemon

A robust, high-performance, deduplicated incremental backup solution built with a hybrid architecture of **Python**, **Rust**, and **Bash**. This tool is designed to provide efficient disk-space usage and reliable data recovery for Linux environments.

---

## Objectives & Job
The **Incremental Backup Daemon** is designed to solve the problem of redundant data storage in backups. Its primary jobs are:
1.  **Deduplication**: Only store unique file contents across multiple backup versions.
2.  **Efficiency**: Use Rust for fast file diffing and metadata management.
3.  **Simplicity**: Provide a straightforward interface for daily backups and point-in-time restores.
4.  **Flexibility**: Allow ignoring specific files/directories via a simple configuration file.

---

## How It Works (Architecture)

The system operates in a **4-phase pipeline** coordinated by a Python orchestrator:

### 1. Scan Phase (Bash)
`bash/scan.sh` walks through the target directory (e.g., your home folder).
-   Checks against `.backupignore`.
-   Calculates SHA-256 hashes for every file.
-   Outputs a snapshot of the current state (`storage/temp/current_state.txt`).

### 2. Diff Engine (Rust)
The Rust engine (`rust/src/main.rs`) compares the current scan against the **latest known snapshot**.
-   Identifies **Added**, **Modified**, and **Deleted** files.
-   Generates a new JSON snapshot in `storage/snapshots/`.
-   Outputs a list of files that actually need to be copied (`changed_files.txt`).

### 3. Copy Phase (Bash)
`bash/copy.sh` processes the changed files list.
-   Moves file contents into `storage/objects/`.
-   **Content-Addressed Storage**: Files are renamed to their SHA-256 hash. If two files have the same content, they share the same object in storage, saving significant space.

### 4. Restore Phase (Bash/Python)
`bash/restore.sh` reconstructs a full directory state from a specific snapshot.
-   Reads the snapshot JSON to find the required hashes.
-   Maps hashes back to original file paths and copies them from the object store.

## How to Operate

### Prerequisites
-   **Python 3.x**
-   **Rust & Cargo**
-   **Bash** (Linux environment)
-   **Dependencies**: `jq` (for JSON processing in shell)

### Running a Backup
Start the orchestrator from the `python/` directory:
```bash
python3 python/orchestrator.py
```
1.  Select `backup` when prompted.
2.  The system will scan your home directory (default) and process changes.

### Restoring a Backup
1.  Check `storage/index.json` or the `storage/snapshots/` directory for the timestamp you wish to restore.
2.  Run the orchestrator:
    ```bash
    python3 python/orchestrator.py
    ```
3.  Select `restore`.
4.  Enter the **timestamp** (e.g., `1714237798`) and the **target directory** for restoration.

-   **Ignore List**: Edit `.backupignore` in the root directory to skip large or temporary directories (e.g., `node_modules`, `.cache`).
-   **Logs**: Check the `logs/` directory for detailed execution history if any step fails.
